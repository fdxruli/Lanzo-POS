OBJETIVO
Se implemento la base cloud de ventas para PRO como read model / snapshot de auditoria. El cobro real sigue siendo local.

LIMITES DE FASE
FREE no cambia. PRO sigue vendiendo local. La venta cloud 6A no descuenta stock, no mueve caja, no crea deuda, no crea abonos, no cancela ventas reales y no sustituye processSaleCore.

SEGURIDAD
RLS quedo activo en las 4 tablas. Se revoco acceso directo anon/authenticated. Las RPCs validan licencia, dispositivo y staff con private.validate_pos_sync_context. Staff requiere permiso pos para subir shadow. Staff sin reports solo consulta sus ventas o las de su dispositivo.

EVENTOS SYNC

INTEGRACION LOCAL
processSaleCore primero confirma executeSaleTransactionSafe. Solo despues del commit local exitoso dispara salesCloudShadowService.syncSaleShadowAfterLocalCommit como fire-and-forget. Si cloud falla, no se revierte la venta. Si el dispositivo esta offline, se marca pendiente y se encola UPSERT_SHADOW.

DEXIE / CACHE
syncDexieBootstrap sube schema a v24 y agrega indices de ventas shadow: cloudSaleId, cloudSalesSyncStatus, cloudSalesLastSyncAt, sourceMode, [sourceMode+timestamp], [cloudSalesSyncStatus+timestamp].
Los snapshots cloud se guardan en sync_cache con prefijo cloud_sale_shadow. Si el snapshot coincide con una venta local, solo se actualizan campos de estado cloud.

QUE NO HACE 6A
No toca pos_products, pos_product_batches, pos_cash_sessions, pos_cash_movements ni pos_customer_ledger. No hay folio cloud definitivo, venta fiada cloud real, devoluciones cloud ni reportes de utilidad definitivos por cloud.
