1. RESUMEN GENERAL

Se reviso la fase 6G con enfoque en estabilidad, consistencia, doble click, offline/online, caja, inventario, credito, cancelaciones, reportes finales y mensajes de usuario final.

Resultado general:
- Se encontro y corrigio un punto critico en frontend/servicio: en PRO cloud, una venta contado compatible podia caer a flujo local/shadow cuando el navegador estaba offline. Eso podia provocar ventas locales no oficiales y riesgo de duplicado al volver online.
- Se corrigio para que PRO cloud offline bloquee ventas cloud reales con mensaje amigable, sin afectar FREE/local.
- Se verifico que las RPCs criticas de 6B, 6C, 6D, 6E, 6E.1 y 6F existen en Supabase.
- Se verificaron tablas criticas y checks estructurales de datos actuales.
- Se verifico que el despliegue Vercel del commit de correccion quedo READY, por lo que el build de produccion paso.

7. OFFLINE / ONLINE / PWA

PRO online:
- Venta cloud permitida si flags y licencia lo habilitan.
- Cancelacion cloud permitida si online y feature activa.
- Reportes cloud final via reportsRepository + reportsCloudRepository.

PRO offline:
- Correccion aplicada en salesCloudCashierService:
  - Antes: venta contado compatible podia caer a local/shadow si offline.
  - Ahora: si PRO cloud cashier esta activo y el metodo es cash/card/transfer/mixed compatible, se bloquea con mensaje friendly.
- Cancelacion cloud ya bloqueaba offline.
- Reportes cloud final ya usan cache/snapshot si existe o fallback local con advertencia no oficial.

FREE/local offline:
- No se tocaron flags FREE/local.
- Si no aplica cloud cashier, el flujo local permanece.

PWA movil:
- La correccion reduce riesgo PWA de venta PRO local accidental al perder conexion.
